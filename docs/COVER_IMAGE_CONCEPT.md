# Cover image concept

## Primary concept
A split-screen product shot:
- left: buyer dashboard showing spend cap, private details card, and staged payouts
- right: mobile runner screen showing accept button, verification badge, and proof submission buttons
- center overlay: simple escrow timeline with scout -> arrival -> heartbeat -> completion

## Visual priorities
- dark UI
- one bright accent color for trust / payout state
- obvious "private until acceptance" badge
- visible cUSD amounts
- one short sentence: "Pay only as proof arrives"

## Avoid
- generic chain diagrams
- token price charts
- protocol-heavy visuals
- too many sponsor logos
