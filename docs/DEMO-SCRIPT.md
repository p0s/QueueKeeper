# Demo script

## Goal
Show the full trust loop in under 4 minutes.

## Steps
1. Open homepage and say the concrete product story: private scout-and-hold procurement for scarce access.
2. Point to the loop ladder and sponsor rail so judges can see Venice, Self, Celo, Uniswap, x402, and ERC-8004 in one screen.
3. Enter **Agent Mode** and show the agent identity card plus the public/private boundary at the top of the composer.
4. Create a `DIRECT_DISPATCH` task with a redacted public brief, private destination, and staged payout ladder.
5. Run the private planner preview and call out the stage path it creates.
6. Post the task and note that the principal can still stop after any verified increment.
7. Open the **Task Command Center** and point to the dominant next-action card, grouped stage ladder, spend boundary, and sponsor tools.
8. Open the runner path, start the Self verification session, and keep the destination visibly locked.
9. Accept only after verification succeeds and show reveal data.
10. Submit a scout proof with an encrypted image bundle.
11. Return to the command center, review the proof, and release the scout increment.
12. Buy one paid x402 venue hint, then trigger an agent decision and explain why it continues or stops.
13. If wallet funding is available, open the optional Uniswap funding tool and capture a normalization receipt.
14. End on `/evidence`, explorer links, and the downloadable `agent.json` / `agent_log.json` artifacts.

## If asked about live chain state
- show Foundry tests
- show the shared escrow ABI export and deployed address file
- toggle the optional live write path in the UI if a wallet is available
- show explorer links for escrow, delegation policy, and any captured tx hashes
- explain that the contract now covers repeated heartbeats, disputes, timeout auto-release, and expiry refunds while the backend still handles encrypted proof media and reveal-token privacy
